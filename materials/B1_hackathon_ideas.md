# ハッカソン テーマアイデア集

> 11:30のテーマ出し＋チーム分けで使う。
> フォーム回答のアイデアが足りない時のリザーブとしても使える。
> 全テーマ共通: **3時間で動くMVP** を作り、16:30にデモする。

### なぜ「自分で作る」のか

外部SaaSを導入したくても、社内のセキュリティ審査が厳しくてなかなか通らない。
だったら **自分で内製すればいい**。AIがあれば3時間でMVPが作れる時代。
今日のハッカソンで作るものは、明日から自分の職場で使えるツールになる。

## テーマ一覧

### 1. PRレビュー要約Bot

**一言**: PRのdiffを食わせたら、レビューポイントを日本語で返すBot

**解決する場面**: PRが大きくなるとレビューに時間がかかる。要点だけ先に知りたい

**MVP（3時間で作るもの）**:
- GitHubのPR URLを入力 → diff取得 → Claude/GPT APIで要約 → 結果表示
- Web UIでもCLIでもOK

**技術スタック**: Python or Node.js + GitHub API + LLM API

**難易度**: ★★☆ 中級（API連携の経験があると楽）

**発展**: GitHub Actionsに組み込んでPR作成時に自動コメント

### 2. 議事録→TODO抽出マシン

**一言**: ミーティングのテキストログから、決定事項・TODO・担当者を自動抽出

**解決する場面**: 会議後に「で、誰が何やるんだっけ？」ってなるやつ

**MVP（3時間で作るもの）**:
- テキストエリアに議事録テキストを貼る
- LLMが構造化: 決定事項 / TODO（担当+期限） / 次回までの宿題
- Markdown or JSON で出力

**技術スタック**: HTML + JS（ブラウザだけで完結可能） + LLM API

**難易度**: ★☆☆ 初級（一番取り組みやすい）

**発展**: Slack連携、Google Docs連携、Issue自動作成

### 3. ドキュメントQ&A Bot（RAG入門）

**一言**: 自分のドキュメントをAIに食わせて、質問したら答えてくれるBot

**解決する場面**: 社内Wiki・READMEが膨大で、欲しい情報が見つからない

**MVP（3時間で作るもの）**:
- Markdownファイル数本をアップロード
- テキスト分割 → ベクトル化 → 類似検索 → LLMで回答生成
- チャットUIで質問 → 回答＋引用元表示

**技術スタック**: Python + LangChain or LlamaIndex + ChromaDB + Streamlit

**難易度**: ★★★ 上級（RAGパイプラインの構築経験がなくても、ライブラリに乗れば3時間でいける）

**発展**: Notion連携、Slack Bot化、社内ナレッジ検索

### デフエンジニアの日常から生まれたアイデア

> 「手話認識AI」のような定番ではなく、デフエンジニアが毎日感じる摩擦から逆算したテーマ。

#### 17. リアルタイム会議マインドマップ

**一言**: 会議の文字起こしから、話の流れ・決定事項・分岐点をリアルタイムでマインドマップ化

**解決する場面**: NotebookLMのマインドマップは便利だが会議後にしか作れない。字幕は流れていくから全体像が掴みにくい。会議中に「今どこの話をしてるか」が一目で分かる地図が欲しい

**MVP（3時間で作るもの）**:
- テキストエリアに議事録テキストを逐次入力（or 文字起こしAPI連携）
- LLMが定期的に（30秒〜1分ごとに）現在のマインドマップを再生成
- トピック・決定事項・未解決をノードで可視化。新しい話題が出たら枝が伸びる

**技術スタック**: HTML + JS + LLM API + マインドマップ描画ライブラリ（D3.js or Markmap）

**難易度**: ★★☆ 中級

**発展**: Zoom/Teams連携でリアルタイム自動生成、会議後にNotebookLMへエクスポート

---

#### 18. "今なに言った？"キャッチャー

**一言**: 会議中、自分の名前が呼ばれた・決定事項が出た・質問された瞬間だけポップアップ通知

**解決する場面**: ビデオ会議で画面共有を見てると字幕が追えない。字幕を見てるとコードが追えない。視線が1つしかない問題

**MVP（3時間で作るもの）**:
- 文字起こしテキストをリアルタイム監視
- キーワード（自分の名前、「決定」「TODO」「質問」等）を検出
- 検出した瞬間、デスクトップ通知 or 画面上にオーバーレイ表示

**技術スタック**: Python or JS + 文字起こしAPI + デスクトップ通知（Notification API）

**難易度**: ★☆☆ 初級

**発展**: 重要度を3段階で色分け、Slack連携で見逃し通知を後から確認

---

#### 19. 会議の"空気"テキスト化

**一言**: 字幕に映らない「笑い」「沈黙」「声の大きさの変化」をメタ情報として可視化

**解決する場面**: 字幕だけだと冗談なのかマジなのか分からない。誰かが怒ってるのか、場が和んでるのかが拾えない

**MVP（3時間で作るもの）**:
- 音声ファイル（会議録音）をアップロード
- 音声分析で「[笑い]」「[沈黙5秒]」「[声のトーン上昇]」を検出
- 字幕テキストにメタ情報をインライン挿入して表示

**技術スタック**: Python + 音声分析API（Hume AI等） + LLMで感情ラベル付け

**難易度**: ★★★ 上級

**発展**: リアルタイム化、感情の推移グラフ表示

---

#### 20. 音の可視化ダッシュボード

**一言**: 環境音をリアルタイムで分類して「誰かが呼んでる」「通知音が鳴った」をアイコン表示

**解決する場面**: オフィスで自分を呼ぶ声、電話、アラーム、ドアノック — 全部聞こえない

**MVP（3時間で作るもの）**:
- ブラウザのマイクで環境音をリアルタイム取得
- 音声分類モデル（YAMNet等）で「人の声」「ノック」「アラーム」「拍手」等を判定
- 検出した音をアイコン+方向で画面に表示するミニダッシュボード

**技術スタック**: JS + Web Audio API + TensorFlow.js（YAMNet） + HTML/CSS

**難易度**: ★★☆ 中級

**発展**: スマートウォッチ連携（振動通知）、音の履歴ログ

#### 21. 非同期スタンドアップBot

**一言**: 朝会を音声なしで完結させる。各自テキスト投稿→AIが要約+依存関係を検出

**解決する場面**: 朝会は音声で回る。字幕対応しても、リアルタイムで全員の発言を追うのは大変

**MVP（3時間で作るもの）**:

- 全員分の回答をLLMが要約
- 「AさんとBさん、同じところで詰まってます」「Cさんの今日のタスクはDさんのブロッカーです」を検出してレポート

**技術スタック**: Python + Slack API（or Discord.js） + LLM API

**難易度**: ★★☆ 中級

**発展**: 週次トレンド分析、チームの健康度スコア

#### 22. ペアプロ仲介AI

**一言**: テキストベースのペアプロを、AIが文脈補完してテンポよくする

**解決する場面**: ペアプロは「声で相談しながら」が前提。テキストチャットだとテンポが遅くて噛み合わない。聴覚障害者だけでなく、リモートワーカーのペアプロ品質も上がる

**MVP（3時間で作るもの）**:
- 2人がそれぞれテキストで意図を入力
- AIが文脈を理解して「こういうことですか？」と補完・整理
- お互いの意図を構造化して並べて表示（左右分割 or ツリー形式）

**技術スタック**: HTML + JS + WebSocket（リアルタイム共有） + LLM API

**難易度**: ★★☆ 中級

**発展**: コードエディタ統合、VS Code拡張化

---

#### 23. 見逃し字幕リプレイ

**一言**: リアルタイム字幕を巻き戻せるようにする。重要発言はAIがハイライト

**解決する場面**: 字幕は流れていく。目を離したら終わり。「今なんて言った？」の巻き戻しができない

**MVP（3時間で作るもの）**:
- リアルタイム字幕テキストをリングバッファに蓄積
- 「直近30秒を再表示」ボタン
- LLMが重要発言（決定事項・質問・数字）を自動ハイライト

**技術スタック**: HTML + JS + 文字起こしAPI + LLM API

**難易度**: ★☆☆ 初級

**発展**: タイムスタンプ付きログ、会議後にサマリー自動生成

#### 24. "聞こえる前提"検出器

**一言**: ドキュメントやUI設計から「音声通知します」等の聞こえる前提を検出し、代替案を提案

**解決する場面**: 仕様書に「電話でお知らせ」「アラーム音で通知」と書いてあっても、聴覚障害者には届かない。設計段階で気づけない

**MVP（3時間で作るもの）**:
- テキストエリアに仕様書・UI文言を貼る
- LLMが「聞こえる前提」の箇所を検出してハイライト
- 各箇所に代替案を提案（例: 「電話通知」→「LINE通知 + バイブレーション」）

**技術スタック**: HTML + JS + LLM API（プロンプト設計だけでいける）

**難易度**: ★☆☆ 初級

**なぜ面白い**: アクセシビリティチェッカーとして全社展開できる。障害者雇用の実績にもなる

#### 25. 動画→構造化ノート変換

**一言**: YouTube動画のURLを入れると、章立て+箇条書き+コードスニペット付きの構造化ノートに変換

**解決する場面**: 技術動画（カンファレンストーク等）の字幕品質はバラバラ。あっても追うのが大変。文字で読みたい

**MVP（3時間で作るもの）**:
- YouTube URL入力 → 字幕テキストを自動取得（YouTube Data API or yt-dlp）
- LLMが「章立て + 箇条書き要約 + コードスニペット抽出」で構造化
- Markdown形式で出力（コピペでObsidian/Notionに貼れる）

**技術スタック**: Python + YouTube字幕取得 + LLM API

**難易度**: ★★☆ 中級

**発展**: 複数動画の横断検索、自動タグ付け
